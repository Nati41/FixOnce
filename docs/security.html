Security Overview

Last updated: March 2026

Summary: FixOnce runs entirely on your local machine. No external servers, no cloud storage, no network traffic outside localhost.

Architecture

Local-Only Design: FixOnce is designed to run 100% locally. The server binds to localhost (127.0.0.1) and does not accept external connections.
Component Communication
Flask Server localhost:5000-5009 only
MCP Server stdio (local process)
Browser Extension localhost only
Dashboard localhost only

Data Storage

Location Contents Access
~/.fixonce/ User data, sessions, config Your user only
.fixonce/ Project decisions, insights Project folder permissions

No External Communication

FixOnce does NOT make any external network calls:

  • No analytics services
  • No telemetry endpoints
  • No update servers (updates are manual)
  • No cloud sync
  • No license validation

Browser Extension Security

If you install the optional browser extension:

Automatic Sanitization: The extension automatically redacts sensitive data before sending to your local server.
Data Type Action
Email addresses Replaced with [EMAIL]
Bearer tokens Replaced with [TOKEN]
Passwords Replaced with [REDACTED]
API keys (32+ chars) Replaced with [API_KEY]
JWT tokens Replaced with [JWT]
Credit card numbers Replaced with [CARD]

Code Changes

FixOnce NEVER modifies your code directly.

All code changes happen through your AI assistant (Claude, Cursor, Codex), which requires your explicit approval before making changes.

  • FixOnce provides context and memory to AI
  • AI proposes changes
  • You review and approve
  • AI makes changes (not FixOnce)

Multi-User Isolation

On shared machines, FixOnce isolates users:

  • Each user has separate ~/.fixonce/ directory
  • Port detection validates ownership
  • Server refuses connections from other users' installations

Reporting Security Issues

  • Open a private security advisory on GitHub
  • Or email the maintainers directly
