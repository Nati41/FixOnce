The ownership model

Memory belongs to the project. Not the AI.

Your memory lives in a .fixonce/ folder alongside the code. Switch agents, close the app, or reinstall FixOnce. The project still knows what it knows.

Private by architecture, not by promise.

Before you install

The short answers.

How is this different from an AI's built-in memory?

Built-in memory belongs to one product. FixOnce belongs to the project, uses explicit engineering structure, and can be loaded by Claude, Codex, Cursor, and other MCP-compatible tools.

Does FixOnce read or upload my entire codebase?

No. FixOnce runs locally and does not upload your code to a FixOnce service. It stores structured project context in the project's own .fixonce/ folder.

Can I choose what gets remembered?

Yes. Proposed decisions, avoids, solved bugs, notes, and next tasks can be reviewed, edited, or rejected before saving.

Which tools are supported?

FixOnce currently works with Claude Code, Codex, Cursor, and other clients that support MCP.
