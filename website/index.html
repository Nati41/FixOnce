Before you install

The short answers.

How is this different from an AI's built-in memory?

Built-in memory belongs to one product. FixOnce belongs to the project. It is demonstrated with Claude, Codex, and Cursor, and is not memory locked to one AI product.

Does FixOnce read or upload my entire codebase?

No. FixOnce runs locally and does not upload your code to a FixOnce service. It stores structured project context in the project's own .fixonce/ folder.

Can I choose what gets remembered?

Yes. Proposed decisions, avoids, solved bugs, notes, and next tasks can be reviewed, edited, or rejected before saving.

Which tools are supported?

The landing page currently demonstrates Claude, Codex, and Cursor. FixOnce is project-owned memory for AI coding tools that can connect to project context.
