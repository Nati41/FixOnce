# FixOnce AI Installation

Install FixOnce on this machine. Follow these steps exactly. Do NOT skip any step. Do NOT silently ignore failures.

## Step 1: Detect Environment

1. Detect OS (macOS, Linux, or Windows)
2. Find Python executable:
   ```bash
   python3 --version
   ```
3. Verify version is >= 3.10

**STOP RULE:** If Python version < 3.10 or Python not found:
- Tell user: "Python 3.10 or newer is required. Install from https://python.org/downloads"
- Do NOT proceed further

## Step 2: Locate FixOnce

Find the FixOnce directory. Check these locations IN ORDER:
- ~/Downloads/FixOnce-main (GitHub ZIP default name)
- ~/Downloads/FixOnce
- ~/Desktop/FixOnce-main
- ~/Desktop/FixOnce
- Current working directory

Use the FIRST directory found that contains requirements.txt.

If not found, ask user: "Where did you extract FixOnce?"

**STOP RULE:** Verify requirements.txt exists:
```bash
ls <fixonce_directory>/requirements.txt
```
If missing, tell user: "requirements.txt not found. FixOnce may be incomplete or corrupted."

Also verify these files exist:
- src/server.py
- src/mcp_server/mcp_memory_server_v2.py

## Step 3: Create Virtual Environment

```bash
cd <fixonce_directory>
python3 -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
```

## Step 4: Install Dependencies

```bash
pip install -r requirements.txt
```

**Import Validation:** After install, verify these imports work:
```python
python3 -c "import flask; import flask_cors; import requests; import fastmcp; print('All imports OK')"
```

If any import fails:
- Report which package failed
- Suggest: `pip install <package_name>`
- Do NOT proceed until all imports succeed

## Step 5: Start Server

```bash
cd <fixonce_directory>/src
python server.py --flask-only &
```

Wait 3 seconds, then verify server is running:
```bash
curl -s http://localhost:5000/api/health || curl -s http://localhost:5001/api/health
```

**Port Handling:**
- Default port is 5000
- If port 5000 is busy, server will try 5001, 5002, ... up to 5009
- Check actual port from server output or:
  ```bash
  cat ~/.fixonce/current_port.txt
  ```
- Remember the actual port for the dashboard URL

## Step 6: Configure MCP (CRITICAL - Follow Exactly)

### MCP CONFIG SAFETY RULES:
- NEVER write `FASTMCP_CHECK_FOR_UPDATES: "false"` - this value is INVALID and breaks MCP
- Allowed values: `"stable"`, `"prerelease"`, `"off"`
- ALWAYS use `"off"` as the default
- ALWAYS create timestamped backup before modifying any config file
- NEVER overwrite unrelated config entries
- If existing JSON/TOML is invalid, STOP and tell user - do not corrupt further

### Backup Command:
```bash
cp <config_file> <config_file>.backup_$(date +%Y%m%d_%H%M%S)
```

### For Claude Code:

Check if CLI exists:
```bash
which claude
```

**If CLI exists (preferred):**
```bash
claude mcp remove fixonce -s user 2>/dev/null
claude mcp add-json fixonce '{
  "command": "<fixonce_dir>/venv/bin/fastmcp",
  "args": ["run", "<fixonce_dir>/src/mcp_server/mcp_memory_server_v2.py", "--transport", "stdio"],
  "env": {
    "PYTHONPATH": "<fixonce_dir>/src",
    "FASTMCP_CHECK_FOR_UPDATES": "off",
    "FASTMCP_SHOW_CLI_BANNER": "false"
  }
}' -s user
```

**If CLI doesn't exist (fallback to file):**
1. Create backup of ~/.claude.json
2. Read existing config (may be empty or not exist)
3. Merge this entry under mcpServers (do NOT overwrite other servers):
```json
{
  "fixonce": {
    "command": "<fixonce_dir>/venv/bin/fastmcp",
    "args": ["run", "<fixonce_dir>/src/mcp_server/mcp_memory_server_v2.py", "--transport", "stdio"],
    "env": {
      "PYTHONPATH": "<fixonce_dir>/src",
      "FASTMCP_CHECK_FOR_UPDATES": "off",
      "FASTMCP_SHOW_CLI_BANNER": "false"
    }
  }
}
```
4. Validate JSON syntax before writing
5. Write updated config

### For Cursor:

1. Create backup of ~/.cursor/mcp.json if it exists
2. Read or create config
3. Merge this entry under mcpServers:
```json
{
  "fixonce": {
    "command": "<fixonce_dir>/venv/bin/fastmcp",

    "env": {

```
4. Validate JSON
5. Write config

### For Codex:

1. Create backup of ~/.codex/config.toml if it exists
2. Add or update TOML section:
```toml
[mcp_servers.fixonce]
command = "<fixonce_dir>/venv/bin/fastmcp"
args = ["run", "<fixonce_dir>/src/mcp_server/mcp_memory_server_v2.py", "--transport", "stdio"]

[mcp_servers.fixonce.env]
PYTHONPATH = "<fixonce_dir>/src"
FASTMCP_CHECK_FOR_UPDATES = "off"
FASTMCP_SHOW_CLI_BANNER = "false"
```

## Step 7: Install Global AI Rules (CRITICAL - DO NOT SKIP!)

**THIS STEP IS MANDATORY. WITHOUT IT, FIXONCE WILL NOT AUTO-INITIALIZE.**

This step makes FixOnce MANDATORY for all future sessions.

1. Create directory if needed:
```bash
mkdir -p ~/.claude
```

2. Copy FixOnce rules to global CLAUDE.md:
```bash
cat <fixonce_dir>/data/global-claude-md.md >> ~/.claude/CLAUDE.md
```

3. **VERIFY the file was created** (DO NOT SKIP THIS CHECK):
```bash
cat ~/.claude/CLAUDE.md | head -5
```

Expected output should start with:
```
# FixOnce Memory System
```

**If the file is empty or doesn't exist, the installation FAILED. Re-run step 2.**

## Step 8: Verify Installation

```bash
curl -s http://localhost:<actual_port>/api/health
```

Should return JSON containing `"status": "ok"` or similar success indicator.

## Step 9: Final Verification Checklist

Before reporting success, verify ALL of these:

```bash
# 1. Server running?
curl -s http://localhost:<actual_port>/api/health | grep -q "ok" && echo "✓ Server OK" || echo "✗ Server FAILED"

# 2. MCP configured?
grep -q "fixonce" ~/.claude.json && echo "✓ MCP OK" || echo "✗ MCP FAILED"

# 3. Global rules installed? (CRITICAL!)
head -1 ~/.claude/CLAUDE.md 2>/dev/null | grep -q "FixOnce Memory System" && echo "✓ Rules OK" || echo "✗ Rules FAILED - MUST FIX!"
```

**If any check shows FAILED, go back and fix it before proceeding.**

## Step 10: Report Final Status

Tell user:

```
FixOnce Installation Complete

Server: http://localhost:<actual_port>
Dashboard: http://localhost:<actual_port>

Verification:
✓ Server running
✓ MCP configured for: [list which editors]
✓ Global AI rules installed (~/.claude/CLAUDE.md)

NEXT: Proceed to Step 11 to authorize the MCP tool.
```

## Step 11: Authorize MCP Tool (CRITICAL - Enables Auto-Start)

**Why this step matters:**
Claude Code blocks new MCP tools until the user explicitly approves them.
Without this approval, FixOnce will NEVER auto-initialize on "hi".

**Action:**
Tell user:
```
⚡ FINAL STEP - Authorize FixOnce

I need to test the FixOnce connection. When I call the tool:
→ You will see a permission prompt
→ Select: "Yes, and don't ask again" (or "Always allow")

This is a ONE-TIME approval. After this, FixOnce will auto-start on every session.

Calling FixOnce now...
```

**Then immediately call:**
```
fo_init(cwd="<current_directory>")
```

**Expected behavior:**
1. Claude Code shows permission dialog
2. User selects "Yes, and don't ask again"
3. Tool executes and returns session info
4. Tell user: "✓ FixOnce authorized! From now on, just say 'hi' in any new session."

**If user sees the tool response WITHOUT a permission dialog:**
That means it was already authorized. Proceed to completion.

## Error Handling Rules

- If ANY step fails, report the exact error message
- Do NOT silently skip failures
- Do NOT modify FixOnce source code
- If JSON/TOML config is already corrupted, tell user and stop
- Always report what was changed and where backups are located
- If port detection fails, tell user to check server output manually
